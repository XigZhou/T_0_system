# 文档与交付要求

## 文档语言

- 数据字典、指标说明、README 新增内容、系统使用说明默认使用中文，除非用户明确要求其他语言。

## 数据文档

- 数据说明文档放在仓库根目录 `docs/` 下，与 `README.md` 同级。
- 每个数据集至少补充以下内容：
  - 来源接口或来源表
  - 输出文件路径
  - 数据粒度与主键
  - 更新逻辑或生成脚本
  - 字段级中文说明
  - 缺失值、停牌、复权处理逻辑

## 指标文档

- 每个指标至少补充以下内容：
  - 指标用途
  - 输入字段
  - 计算公式
  - 窗口参数
  - 边界条件
  - 一个带数字的计算示例

## 回测系统交付

- 面向交付的回测系统默认保留前端入口，除非用户明确要求 CLI-only。
- `README.md` 需要覆盖准备工作、环境变量、数据准备、启动方式、核心功能入口、结果复现方式与输出位置。
- 如果仓库中已有 `scripts/verify_delivery.py`，优先先跑它，再补充更具体的测试和冒烟验证。
- 不要只说明“服务能跑起来”，还要说明如何复现结果、输出在哪、怎么校验。

## 交付门槛

- 对外交付前先跑和本次改动直接相关的自动化测试。
- 如果触及 API、前端或回测主流程，额外做一次本地启动或功能冒烟验证。
- 只有在仓库真实可推送、且用户要求该步骤时，才能声称 GitHub 交付已完成。
