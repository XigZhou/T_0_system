# Tushare 默认约定

## 默认规则

- 默认从本机环境变量 `TUSHARE_TOKEN` 读取 token。
- 除非环境变量缺失且任务被阻塞，否则不要要求用户再次粘贴 token。
- 不要在终端输出、日志、代码、文档或提交记录中打印、回显或写入 token 明文。

## 权限假设

- 默认按 2000 积分权限规划实现，除非用户另有说明。
- 优先使用该权限下通常可用的接口。
- 如果某个接口是否可用不确定，先明确写出假设，再继续实现，而不是停在原地等待。

## 数据处理

- 优先把 token 读取逻辑集中到配置层或辅助模块，避免散落在多个脚本里。
- 原始拉取、处理后数据、衍生指标在代码和文档中要尽量分层清楚。
- 当正确性依赖于数据更新时间、交易日历对齐、前复权或除权逻辑时，要明确写出这些处理方式。
